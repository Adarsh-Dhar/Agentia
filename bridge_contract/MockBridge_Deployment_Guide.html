🤔Why a Mock Bridge?

Real Initia bridging uses IBC — Inter-Blockchain Communication. A live bridge call requires a massive payload including cryptographic Merkle proofs, destination chain IDs, sequence numbers, and timeout heights. Building that payload by hand is complex; if the blockchain rejects it, you won't know whether your bot logic is broken or your bridge math is wrong.

A Mock Bridge isolates the variable. It's a real on-chain contract that accepts the exact same function signature as the real bridge — so your bot's payload is 100% valid — but the contract body does nothing. One successful transaction proves end-to-end signing, gas estimation, address resolution, and broadcast all work correctly.

✅Prerequisites

Before you begin, make sure you have all of the following ready:

  • initiad CLI installed and in your PATH (download here)
  • A test wallet created with initiad keys add test-account
  • Your wallet funded with testnet uinit tokens (use the Initia faucet)
  • Your init1... wallet address (run initiad keys show test-account -a)
  • Your 0x... hex wallet address (run initiad keys show test-account --output json)
1
Initialize the Move Project

Open a terminal and run these commands to scaffold a new Move project:

bash 
mkdir mock_bridge
cd mock_bridge
initiad move init --name MockBridge

This generates the standard Move project layout:

mock_bridge/ ├── sources/ # Your .move source files go here ├── Move.toml # Package manifest (edit in Step 2) └── .gitignore # Auto-generated
2
Configure Move.toml

Open Move.toml and replace its contents with the following. The deployer = "_" placeholder is filled in automatically when you run the deploy command with --named-addresses.

Move.toml 
[package]
name = "MockBridge"
version = "0.0.0"

[addresses]
deployer = "_"

[dependencies]
InitiaStdlib = { git = "https://github.com/initia-labs/initia.git", subdir = "api/move/ext/initia_stdlib", rev = "main" }
ℹ️ The InitiaStdlib dependency is cloned from GitHub during compilation. Make sure you have internet access and git installed.
3
Write the Bridge Contract

Inside sources/, create a file named bridge.move with the following content:

sources/bridge.move 
/// MockBridge — interwoven_bridge module
///
/// Simulates an IBC/L1 bridge for local testing.
/// Accepts a sweep call, does nothing, returns success.

module deployer::interwoven_bridge {
    use std::signer;

    /// Mock bridge entry point.
    ///
    /// <CoinType> lets your bot pass any token denom
    /// (USDC, INIT, IBC-wrapped assets) without contract changes.
    ///
    /// On a real bridge this would lock tokens and emit an IBC packet.
    /// Here it does nothing — that's the point. ✓
    public entry fun sweep_to_l1<CoinType>(
        _account: &signer,
        _amount: u64
    ) {
        // Intentionally empty.
        // A successful no-op proves your bot's payload is correct.
    }
}
💡 The underscore prefix on _account and _amount suppresses "unused variable" compiler warnings. The contract compiles cleanly with zero warnings.
4
Deploy to Testnet

Run the publish command from inside the mock_bridge/ directory. Replace <YOUR_WALLET_ADDRESS> with your actual init1... address:

bash 
initiad tx move publish . \
    --named-addresses deployer=<YOUR_WALLET_ADDRESS> \
    --from test-account \
    --gas auto \
    --gas-adjustment 1.5 \
    --gas-prices 0.015uinit \
    --node https://rpc.testnet.initia.xyz:443 \
    --chain-id initiation-2
⚠️ If your tokens are on the older testnet, use --chain-id initiation-1 instead of initiation-2.

The CLI will ask you to confirm the transaction. Type y and press Enter. You'll see JSON output like this:

output (abbreviated)
{
  "txhash": "ABC123...",
  "code": 0,
  "raw_log": "...",
  "events": [
    {
      "type": "move",
      "attributes": [
        {
          "key": "type_tag",
          "value": "ModulePublishedEvent"
        },
        {
          "key": "data",
          "value": "{ \"address\": \"0xYOUR_MODULE_ADDRESS\" }"
        }
      ]
    }
  ]
}

Find the ModulePublishedEvent entry and copy the 0x... address. This is your bridge contract address.

5
Hook it up to Your Bot

Open your bot's .env file and add these two lines:

.env 
# Address from the ModulePublishedEvent in Step 4
INITIA_BRIDGE_ADDRESS=0x<address_from_event>

# Your wallet's 0x hex address (NOT the init1... bech32 form)
USER_WALLET_ADDRESS=0x<your_wallet_hex_address>
ℹ️ To get your 0x hex address: run initiad keys show test-account --output json and look for the "address" field, or convert your init1... address using a Bech32 decoder.

Now restart your bot:

bash 
npm run start

🎯Verifying Success

✓ You're done when you see this in your bot's log:

[ACT] ✓ Atomic execution succeeded

This confirms that your bot correctly constructed the sweep_to_l1 transaction, estimated gas, signed with your key, and broadcast to the chain — all without a real IBC relay involved.

🔧Troubleshooting

?
Common Issues & Fixes

❌ "account not found" error

Your wallet hasn't received any testnet tokens yet. Go to faucet.testnet.initia.xyz and paste your init1... address.

❌ "insufficient fees" error

Try increasing --gas-adjustment to 2.0 or setting --gas to a fixed number like 500000.

❌ "wrong chain-id" error

Run initiad status --node https://rpc.testnet.initia.xyz:443 | grep chain_id to confirm the current testnet ID and update your command accordingly.

❌ Bot can't find the module

Double-check that INITIA_BRIDGE_ADDRESS in your .env starts with 0x and matches the address from ModulePublishedEvent exactly (not your init1... address).

❌ Compilation fails with "unresolved dependency"

The InitiaStdlib Git dependency needs network access. Make sure you're online, git is installed, and GitHub isn't blocked on your network.
